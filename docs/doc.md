## 1. Введение и философия

**LuminaEngine** — это легковесный игровой 3D-движок, написанный на JavaScript с использованием библиотеки **THREE.js** для рендеринга. Движок спроектирован на основе популярного архитектурного паттерна **Entity-Component-System (ECS)**, что делает его гибким и расширяемым.

Основная философия движка — простота и модульность. Вместо monolithic-классов, вся игровая логика разделяется на небольшие, переиспользуемые **Компоненты**, которые "навешиваются" на пустые **Игровые Объекты**.

### Основные принципы:
*   **Игровой Объект (GameObject):** Это "контейнер" в игровом мире. Сам по себе он ничего не делает, кроме как существует в определённой точке пространства (`transform`). Это может быть игрок, враг, стена, пуля и т.д.
*   **Компонент (Component):** Это "поведение" или "свойство" игрового объекта. Компоненты содержат всю логику. Например, `PlayerController` (логика управления игроком), `MeshRenderer` (логика отрисовки модели), `Collider` (логика столкновений).
*   **Движок (Engine):** Это центральный "оркестр", который управляет всем: инициализирует систему, запускает игровой цикл (game loop), обновляет все объекты и компоненты, а также управляет подсистемами (рендеринг, ввод).

## 2. Основные концепции

### The Game Loop (Игровой цикл)

Сердце любого игрового движка. В `LuminaEngine` он реализован в методе `Engine.gameLoop`. На каждом кадре (frame) происходит следующий цикл:

1.  **Расчет `deltaTime`**: Вычисляется время, прошедшее с предыдущего кадра. Это критически важно для создания физики и движения, не зависящих от частоты кадров (FPS).
2.  **Обновление всех `GameObject`**: Движок проходит по списку всех игровых объектов и вызывает их метод `update(deltaTime)`.
3.  **Обновление Компонентов**: Метод `GameObject.update()` в свою очередь вызывает методы `update(deltaTime)` у всех своих компонентов. Именно здесь происходит вся игровая логика (перемещение, стрельба, AI и т.д.).
4.  **Рендеринг**: После обновления состояния всех объектов, `Renderer` отрисовывает текущую сцену на экране.
5.  **Обновление UI**: Отображается отладочная информация (FPS, координаты).
6.  **Сброс ввода**: Дельта движения мыши сбрасывается, чтобы на следующем кадре считывать уже новое смещение.
7.  **Планирование следующего кадра**: `requestAnimationFrame()` говорит браузеру, что мы готовы к отрисовке следующего кадра.

### Математика и система координат

*   **Система координат**: LuminaEngine использует правую систему координат, унаследованную от THREE.js (и OpenGL):
    *   **+X**: Вправо
    *   **+Y**: Вверх
    *   **-Z**: Вглубь экрана (вперед)
*   **`deltaTime`**: Время в секундах, прошедшее с последнего кадра. Все движения должны умножаться на `deltaTime`, чтобы скорость была постоянной, независимо от того, работает игра при 30 FPS или 144 FPS.
    *   Пример: `position.x += speed * deltaTime;` // Правильно!
    *   Пример: `position.x += speed;` // Неправильно, скорость будет зависеть от FPS.
*   **Вращение (Rotation)**: Вращение в THREE.js измеряется в **радианах**. Для удобства отображения в UI они переводятся в градусы (`rotation * 180 / Math.PI`).
    *   **Yaw (Рыскание)**: Поворот вокруг оси Y (поворот головы влево-вправо). Управляется вращением `GameObject.transform`.
    *   **Pitch (Тангаж)**: Поворот вокруг оси X (взгляд вверх-вниз). Управляется вращением `Camera`, которая является дочерним объектом по отношению к `GameObject` игрока. Это стандартный подход для FPS-камер, чтобы избежать "заваливания" камеры при одновременном повороте и наклоне.

## 3. API Reference (Справка по классам)

### `Engine.js`

Центральный класс, управляющий всем приложением.

| Свойство/Метод             | Описание                                                                                                                              |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `constructor(canvasId)`    | Инициализирует `Renderer`, `InputManager`, создает пустой список игровых объектов.                                                     |
| `renderer`                 | Экземпляр класса `Renderer`.                                                                                                          |
| `inputManager`             | Экземпляр класса `InputManager`.                                                                                                      |
| `setPlayer(gameObject)`    | Устанавливает переданный `GameObject` в качестве игрока. Это нужно для `Renderer`, чтобы он мог отображать его координаты в UI.        |
| `addGameObject(gameObject)`| Добавляет `GameObject` в движок. При этом объект добавляется в список на обновление и его `transform` добавляется в сцену THREE.js.   |
| `start()`                  | Запускает игровой цикл. Перед этим вызывает метод `start()` у всех добавленных `GameObject` для однократной инициализации.               |
| `gameLoop()`               | Основной метод, выполняющийся каждый кадр. Описан выше.                                                                               |

---

### `GameObject.js`

Контейнер для компонентов, представляющий объект в игровом мире.

| Свойство/Метод                             | Описание                                                                                                                            |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| `constructor(name)`                        | Создает объект с именем и `THREE.Object3D` в качестве `transform`.                                                                    |
| `transform`                                | Экземпляр `THREE.Object3D`. Содержит `position`, `rotation`, `scale` объекта. Все дочерние объекты (например, камера) движутся вместе с ним. |
| `addComponent(ComponentClass, ...args)`    | Создает экземпляр класса `ComponentClass`, передавая ему себя (`this`) и все остальные аргументы (`...args`). Добавляет компонент к объекту. Возвращает созданный компонент. |
| `getComponent(ComponentClass)`             | Находит и возвращает первый компонент указанного класса, прикрепленный к этому объекту. Полезно для взаимодействия между компонентами. |
| `start()`                                  | Вызывается движком один раз перед первым кадром. Делегирует вызов всем своим компонентам.                                            |
| `update(deltaTime)`                        | Вызывается движком каждый кадр. Делегирует вызов всем своим компонентам, передавая `deltaTime`.                                       |

---

### `Component.js`

Базовый класс для всей игровой логики. **Этот класс не используется напрямую, от него нужно наследоваться.**

| Свойство/Метод        | Описание                                                                                                   |
| --------------------- | ---------------------------------------------------------------------------------------------------------- |
| `constructor(gameObject)` | Принимает родительский `GameObject`, сохраняет ссылки на него и на его `transform` для удобства доступа. |
| `gameObject`          | Ссылка на `GameObject`, к которому прикреплен компонент.                                                   |
| `transform`           | Удобная ссылка на `gameObject.transform`.                                                                  |
| `engine`              | Ссылка на главный `Engine`. Устанавливается движком автоматически.                                           |
| `start()`             | **(Переопределяемый)**. Вызывается один раз перед первым кадром. Используется для инициализации (например, поиска других компонентов). |
| `update(deltaTime)`   | **(Переопределяемый)**. Вызывается каждый кадр. Здесь должна находиться вся основная логика компонента.     |

---

### `InputManager.js`

Управляет вводом с клавиатуры и мыши, включая режим "захвата" курсора (Pointer Lock).

| Свойство/Метод      | Описание                                                                                                                                                               |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `constructor(targetElement)` | Принимает DOM-элемент (обычно `<canvas>`), по клику на который будет запрашиваться Pointer Lock. Настраивает все слушатели событий.                               |
| `keys`              | Объект, хранящий состояние клавиш. Например, `keys['KeyW']` будет `true`, если W зажата.                                                                                  |
| `mouseDelta`        | Объект `{x, y}`, накапливающий смещение мыши с момента последнего сброса.                                                                                                |
| `onKeyDown(event)`  | Обработчик нажатия клавиши. Если включен Pointer Lock, он отменяет действие браузера по умолчанию (`event.preventDefault()`) для большинства клавиш, чтобы игра не "дергалась". |
| `isKeyDown(key)`    | Возвращает `true`, если клавиша с кодом `key` (например, `'Space'`, `'KeyA'`) в данный момент зажата.                                                                    |
| `getMouseDelta()`   | Возвращает накопленное смещение мыши за кадр.                                                                                                                          |
| `resetMouseDelta()` | Сбрасывает накопленное смещение мыши в 0. Вызывается движком в конце каждого кадра.                                                                                    |

---

### `Renderer.js`

Отвечает за всю графическую часть, используя THREE.js.

| Свойство/Метод              | Описание                                                                                                                                           |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `constructor(canvasId)`     | Инициализирует сцену THREE.js, камеру, рендерер и настраивает их. Находит DOM-элементы для вывода UI.                                                |
| `domElement`                | Геттер, возвращающий `<canvas>` элемент, который используется для рендеринга. Нужен для `InputManager`.                                               |
| `render()`                  | Вызывает `renderer.render(scene, camera)`, отрисовывая текущее состояние сцены.                                                                     |
| `updateUI(fps, playerTransform)` | Обновляет текстовые поля на экране. Отображает FPS и данные о положении и вращении игрока (`playerTransform`).                                      |

---

### Физика: `PhysicsEngine.js`, `RigidBody.js`, `Colliders.js`

Простая AABB-физика: гравитация + разрешение столкновений между `RigidBody`. Сама физика не привязана к рендерингу и не знает про воксельные миры — это отдельная, переиспользуемая подсистема движка.

**`RigidBody`** (компонент) — навешивается на `GameObject`, который должен участвовать в физике.

| Свойство/Метод                  | Описание                                                                                                    |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `constructor(gameObject, options)` | `options.bodyType`: `'dynamic'` (подвержен гравитации и толкается при столкновении) или `'static'` (неподвижен, например пол или стена). |
| `velocity`                       | `THREE.Vector3` — текущая скорость тела; `PhysicsEngine` прибавляет к ней гравитацию и применяет к позиции.  |
| `isGrounded`                     | `true`, если в текущем кадре тело опирается на что-то снизу (используется, например, для прыжка).             |
| `start()`                        | Регистрирует тело в `engine.physicsEngine`.                                                                   |

**`Collider` / `BoxCollider` / `HeightfieldCollider`** (`Colliders.js`) — форма тела для столкновений.

| Класс                 | Описание                                                                                                                       |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `Collider`             | Базовый класс, задаёт только `type`. Не используется напрямую.                                                                  |
| `BoxCollider(size)`    | Прямоугольный параллелепипед (AABB) с полуразмерами `halfSize`, вычисленными из `size`.                                          |
| `HeightfieldCollider(geometry, segmentsX, segmentsZ)` | Коллайдер под ландшафт из `THREE.PlaneGeometry`: строит сетку высот из геометрии и даёт `getHeightAt(worldX, worldZ)` — билинейно интерполированную высоту поверхности в произвольной точке. |

**`PhysicsEngine`** — считает и разрешает столкновения каждый кадр.

| Свойство/Метод                       | Описание                                                                                                                    |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `gravity`                              | `THREE.Vector3`, по умолчанию `(0, -9.81, 0)`.                                                                             |
| `addRigidBody(body)`                   | Регистрирует тело в списке физики (вызывается автоматически из `RigidBody.start()`).                                        |
| `update(deltaTime)`                    | Применяет гравитацию ко всем `dynamic`-телам, сдвигает их по скорости, затем несколько итераций разрешает столкновения.     |
| `resolveAABBCollision(bodyA, bodyB, ...)` | Столкновение box-vs-box: находит ось наименьшего перекрытия и раздвигает тела вдоль неё (пропорционально, если оба `dynamic`). |
| `resolveBoxVsHeightfield(...)`         | Столкновение box-vs-ландшафт: проверяет 4 нижних угла коллайдера против высоты поверхности и выталкивает тело наверх при проникновении. |

## 4. Как это работает вместе: Пример создания игрока

Допустим, мы хотим создать управляемого игрока.

**1. Создаем компонент `PlayerController.js`:**

```javascript
// Lumina/js/game/components/PlayerController.js
import { Component } from '../../core/Component.js';

export class PlayerController extends Component {
    constructor(gameObject, speed = 5, sensitivity = 0.002) {
        super(gameObject);
        this.speed = speed;
        this.sensitivity = sensitivity;
        this.camera = this.engine.renderer.camera; // Получаем камеру
        
        // Привязываем камеру к игроку, чтобы она двигалась вместе с ним
        this.transform.add(this.camera); 
    }

    start() {
        // Начальная настройка, если нужна
    }

    update(deltaTime) {
        const input = this.engine.inputManager;

        // Вращение (Yaw/Pitch)
        const mouseDelta = input.getMouseDelta();
        // Вращаем игрока по Y (влево/вправо)
        this.transform.rotation.y -= mouseDelta.x * this.sensitivity;
        // Вращаем камеру по X (вверх/вниз)
        this.camera.rotation.x -= mouseDelta.y * this.sensitivity;
        
        // Ограничиваем вертикальный обзор
        this.camera.rotation.x = Math.max(-Math.PI / 2, Math.min(Math.PI / 2, this.camera.rotation.x));
        
        // Движение (WASD)
        const moveDirection = new THREE.Vector3();
        if (input.isKeyDown('KeyW')) moveDirection.z -= 1;
        if (input.isKeyDown('KeyS')) moveDirection.z += 1;
        if (input.isKeyDown('KeyA')) moveDirection.x -= 1;
        if (input.isKeyDown('KeyD')) moveDirection.x += 1;

        if (moveDirection.lengthSq() > 0) {
            moveDirection.normalize();
            // Применяем вращение игрока к направлению движения
            moveDirection.applyQuaternion(this.transform.quaternion); 
            
            this.transform.position.add(moveDirection.multiplyScalar(this.speed * deltaTime));
        }
    }
}
```

**2. Используем его в основном файле игры (`main.js`):**

```javascript
// Lumina/js/game/main.js
import { Engine } from '../core/Engine.js';
import { GameObject } from '../core/GameObject.js';
import { PlayerController } from './components/PlayerController.js';

// 1. Создаем движок
const engine = new Engine('game-canvas');

// 2. Создаем игровой объект для игрока
const player = new GameObject('Player');
player.transform.position.set(0, 1.7, 5); // Начальная позиция

// 3. Добавляем компонент контроллера к игроку
// Передаем скорость и чувствительность как аргументы
player.addComponent(PlayerController, 5, 0.002);

// 4. Добавляем игрока в движок
engine.addGameObject(player);
engine.setPlayer(player); // Указываем, что это наш главный игрок

// 5. ... (создание мира, других объектов) ...

// 6. Запускаем игру
engine.start();
```

Этот пример демонстрирует всю мощь компонентной архитектуры:
*   `GameObject` — это просто точка в пространстве.
*   `PlayerController` добавляет всю логику движения и обзора.
*   `Engine` всё это связывает и заставляет работать в цикле.
